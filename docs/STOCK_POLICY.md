# Политика движения по складам

## Два склада (v1)

| Склад | Типы товаров (префикс) |
|-------|-------------------------|
| **Склад комплектующих** | CP, SP |
| **Склад готовых изделий** | FG, BD |

Услуги (SV) и прочие начисления (CH) **не ведут** складской учёт.

## Цепочка заказа (практика עוסק פטור)

```
הצעת מחיר (Q-)  →  הזמנה  →  ביצוע (работа)  →  חשבון חיוב / חשבון עסקה (H-)
  →  המתנה לתשלום  →  תשלום בבנק  →  קבלה (R-)  →  סגור
```

Для **עוסק פטור** документ **חשבון עסקה** и **חשבון חיוב** в учёте ведут себя одинаково: это требование оплаты после выполнения работ, без מע״מ. В системе v1 используется серия **H-** (חשבון חיוב).

## Когда остатки **не** меняются

| Событие | Движение по складу |
|---------|-------------------|
| **הצעת מחיר** (Q-) | **Нет** |
| Принятие **הזמנה**, правки строк | **Нет** |
| Статус **«в работе»** (ביצוע — монтаж, сборка) | **Нет** — работа ещё не оформлена счётом |
| Фиксация оплаты / **קבלה** (R-) | **Нет** — только деньги, склад уже списан |

## Когда остатки **меняются**

| Операция | Когда | Склад |
|----------|--------|--------|
| **Приход** | Ручной приход, импорт | По типу товара |
| **Корректировка** | Правка остатка в каталоге | По типу товара |
| **Расход (продажа)** | Выписка **חשבון חיוב / חשבון עסקה** (H-) после выполнения | FG/BD + BOM → см. ниже |

**Списание со склада — в момент выписки H-**, а не при הצעת מחיר, не при «начале работ» и не при קבלה.

Логика: к моменту H- работа выполнена, товар/услуга переданы клиенту в смысле выставления счёта; קבלה подтверждает только оплату.

## Продажа готового изделия с BOM

При выписке H- по строке **FG** или **BD**:

1. Со **склада готовых** — N шт. готового товара.
2. Со **склада комплектующих** — по BOM: `N × количество` для каждого CP/SP.

## API (v1, до полного модуля документов)

| Действие | Endpoint | Склад |
|----------|----------|--------|
| Начать выполнение (без склада) | `POST /api/orders/{id}/start-work` | Нет |
| חשבון חיוב (H-) + списание | `POST /api/orders/{id}/issue-charge-invoice` | Да, однократно |
| Приход | `POST /api/warehouse/receipt` | Да |

## Резервирование (будущее)

Резерв под заказ при принятии הזמנה — не в v1.
